# 判断标准（Criteria）

## 值得提炼为 Episodic 的条件（满足其一即可）

| 条件 | 检查字段 | 说明 |
|------|---------|------|
| **有决策** | `决策/犹豫` | 做出了影响后续方向的选择 |
| **有教训** | `教训` | 学到了什么 / 发现了什么不对 |
| **有产出** | `结果` 中的 `[[...]]` 引用 | 创建了重要的文档/代码/设计 |
| **有冲突** | `决策/犹豫` 中的用户纠正 | 用户的反馈和预期不一致 |
| **有重复** | `建议提炼` 中的重复提示 | 同一件事第三次做了 |

## 不值得提炼的

- 常规问答（"帮我改个文件名"）
- 纯探索性操作（"帮我看看这个文件"）
- 纯执行操作（用户给了完整指令，Agent 只是打字）

## Procedural 沉淀标准

| 条件 | 说明 |
|------|------|
| 同一方法成功 3+ 次 | 跨多个 Episodic 识别到同一做法有效 |
| 明确优化过的流程 | 某个流程被证明比之前更好 |

## 分流判断标准

| 判断问题 | 去向 |
|---------|------|
| 这个模式是关于"怎么做"的技巧/流程/判断吗？ | → Procedural（用户确认后写入） |
| 这个模式是关于"是什么"的概念/事实/结构吗？ | → Semantic |
| 两者都有？ | → 分别写入两边 |

**一句话**：Episodic 是分流点，"怎么做"去 Procedural，"是什么"去 Semantic，Semantic 不是 Procedural 的必经之路。

## L5 Identity 信号检测标准

以下信号来自 session 中的**用户自我描述**或**反思性表述**，属于 Identity 级内容：

| 信号类型 | 检测特征 | 示例 |
|----------|----------|------|
| 角色变化 | 角色/职责描述的语义变化 | "我现在主要负责..."、"我转型做了..." |
| 能力变化 | 技能掌握声明 + 跨 session 复现 | "我已经会..."、同类任务出现 3+ 次 |
| 价值观变化 | 偏好/优先级声明 | "我更看重..."、"优先考虑..."、"不再..." |
| 自我认知 | 反思性/自我评估语句 | "我发现自己擅长..."、"我不适合..." |

### L5 触发条件

| 条件 | 说明 |
|------|------|
| session 中有 `用户自我描述` 字段且非空 | 直接作为 L5 候选信号 |
| `决策/犹豫` 中包含用户偏好/角色变化 | 作为补充信号（如 `用户自我描述` 为空） |

### 不值得触发 L5 的

- 情绪性抱怨（"我今天好累"、"这也太难了"）
- 临时状态描述（"这周很忙"、"最近没时间"）
- 无自我指涉的技术讨论（"这个 API 很难用" ≠ "我不擅长 API 设计"）

### 去重规则

- 同一信号（核心语义相同，仅措辞差异）在 30 天内已有 pending 条目时，不重复生成
- 已 rejected 的信号 7 天内不重新提议
